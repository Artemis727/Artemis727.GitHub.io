需求沟通

  1. 需求收集（5+2）

    明确：为什么、做什么、谁做、何时、何地、如何、多少。

  2. 需求分析（8）

    • 需求产生的背景和原因
    • 需求的目的和程度（最好可以量化）
    • 使用人员和场景
    • 会影响到哪些业务领域
    • 统计指标和维度
    • 展现形式
    • 数据源
    • 交付形式和标准

  3. 目标对齐

    梳理整个流程，把整个流程中的关键环节都列出来，这是其实运用了MVC（最小可行性成品）的思想，每个环节都和业务同学碰一下，保持信息共享，确保需求的进行没有跑偏就好，即使有问题影响也不大，及时发现进行调整。

  4. 信息同步

    整个流程一定要数字化，保持信息畅通无阻。信息共享真的很重要，让业务同学清晰看见自己需求的进度，进行得如何了，会让业务同学具有安全感。
